<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added real-time collaboration: Multiple users edit documents simultaneously with live cursors and conflict resolution.
- Introduced advanced search: Filter by tags, dates, and content types with full-text indexing.

## Improvements

- Enhanced MDX rendering performance by 40% for large documentation sites.
- Updated UI components with better accessibility support, including ARIA labels.

## Bug Fixes

- Fixed rendering issues with nested code blocks in MDX files.
- Resolved authentication errors during session timeouts.

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "bugfix"]}>

## New Features

- Custom themes: Apply your brand colors like `#3B82F6` directly in settings.
- Version history: Track and revert document changes with diff views.

## Bug Fixes

- Patched cross-site scripting vulnerability in user-generated content.
- Corrected table rendering in exported PDFs.

## Improvements

- Optimized image loading for faster page renders.

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["breaking", "feature"]}>

## New Features

- Core MDX documentation engine with component support.
- Project organization: Create spaces, folders, and tags for your docs.

## Breaking Changes

- Updated API endpoints from `/v0/` to `/v1/` prefix. Migrate using the guide.

## Initial Setup

- Basic authentication with API keys.
- Dashboard for managing documentation spaces.

</Update>

## Stay Updated

Follow these steps to monitor and apply updates:

<Steps>
  <Step title="Check Dashboard" icon="monitor">
    Visit your Bk Documentation dashboard daily for banners on new releases.
  </Step>
  <Step title="Review Notes" icon="book-open">
    Read release notes here and test changes in a staging space.
  </Step>
  <Step title="Update Config" icon="settings">
    Run the update command:

    <CodeGroup tabs="CLI,Docker">
```bash

```
```dockerfile

docker-compose up -d
```
    </CodeGroup>
  </Step>
</Steps>
